# Design & marketing review

A standing critique pass over the public face of the repo (README = landing page, docs = funnel, CLI
= product UI). Severity: 🔴 must-fix · 🟡 should-fix · 🟢 polish. Update as the repo evolves.

## First impression (README)

- 🟢 Leads with the felt problem ("one model … when it's wrong, it's wrong everywhere at once") and
  the tagline, not an academic definition.
- 🟢 **Hero = a contested-verdict terminal demo.** The single highest-ROI element: it shows the one
  thing a single model can't — five lenses, a verdict, and **preserved dissent**. Keep it above the
  fold and keyless (`--demo`).
- 🟢 A comparison table positions it against single-agent / human committee / generic multi-agent
  frameworks so reviewers don't miscategorise it as "another orchestrator."
- 🟡 Keep the badge row honest (aware/aligned, never compliant/certified).

## Positioning & differentiation

- 🟢 **Honesty merchandised, not buried.** "Councils can be wrong → a human owns the critical call"
  and "no lock-in / survives a provider cut-off" sit in the body as differentiators, not footnotes.
- 🟢 A "How this relates to Elder Mind" box prevents brand confusion (govern the *action* vs govern
  the *decision*; they compose).
- 🟡 Lead cyber; the domain-adaptability angle is a strength for the methodology but a dilution risk
  for the launch headline — keep it a footnote.

## Usability & onboarding

- 🟢 `START-HERE.md` routes by persona; row one is a time-boxed "see a council decide in 5 minutes".
- 🟢 One-job-per-doc: METHODOLOGY is the reference, CONCEPT is the on-ramp; the routing table lives in
  METHODOLOGY only (doc-as-test enforces it).
- 🟢 GLOSSARY carries the two predictable confusions (lenses ≠ councils; council-mode ≠ risk-routing).

## Product UI (the CLI)

- 🟢 Every `convene` prints the verdict, each lens's vote, the **preserved dissent**, the route, and
  the decision id — colourised, `NO_COLOR`-aware. The output *is* the teaching moment.
- 🟡 `--demo` is labelled "deterministic sample votes (keyless, CI-safe)" so newcomers don't think it
  is only for the test suite.

## Consistency

- 🟢 Brand palette + voice codified in [BRAND.md](BRAND.md); claims held at the honest ceiling and
  CI-enforced; `eldercouncil` naming with no stray variants.

## Open polish (tracked)

- 🟡 Branded PDFs + social card + demo GIF are generated by `tools/build_*.py`; ensure the social
  preview is uploaded at launch (a GitHub UI step).
- 🟢 Diagrams are inline mermaid (diffable, free); only the banner/social/GIF are binary.
